## 1주차 과제 회고

1주차 과제로 도서관 시스템을 아주 간단히 구현했다. 

---

### Book 클래스 (상속)

가장 중요한 책을 관리하는 부분을 상속을 이용해서 관리했다. 최상단의 부모 클래스로 `Book 클래스`를 생성하고, `BookName`을 변수로, `BookList()` 로 책 이름을 출력하는 메소드를 구현했다. 

이후에 책의 형태를 종이,전자로 나누어 `PaperBook`,`EBook` 클래스를 생성해 `Book클래스`를 상속받았다. 그리고 책의 분류도 형태에 따라 나눠서 클래스를 생성해 각각 `PaperBook` 과 `EBook` 를 상속받았다. 이 과정에서 책 이름을 출력하는 메소드를 각 클래스별로 두지 않기 위해, `super()` 를 사용하여 `CartoonBooks` 등의 클래스도 `Book클래스`의 `BookList()` 메소드를 호출할 수 있게 하였다. 

이 과정에서 클래스 간의 부모,자식 관계 그리고`super()`의 사용에 대해 잘 이해할 수 있게 되었다. 

하지만, 이렇게 상속을 사용하면서 각 클래스 별로 크게 기능이 있지 않다면 굳이 상속을 사용해야 할까 라는 생각이 들었고, `PaperBook` 클래스에 있는 `BookForm` 변수나, `CartoonBook` 클래스에 있는 `BookFeild` 변수를 제대로 사용하지 못해 아쉬움도 남았다.

### Library 클래스

- `LibraryCode` 변수는 변하지 않는 값이기 때문에, `static`으로 선언
- `LiteratureBook` 등과 같은 2차 상속을 받은 클래스를 배열로 선언

#### VerifyCode 메소드

일반 도서관은 사용자가 등록이 되어있지만, 그렇게 까지 구현하긴 힘들거 같아서 인증코드를 입력하는 방식으로 구현했다. 

#### BookListCreate 메소드

책의 분류에 따라서 책의 개수를 정하고, 각 객체에 책 이름을 할당했다. 

#### BookListViewForm 메소드

책 목록을 보여주기 위해 책의 형태를 선택하는 메소드이다. 

#### BookListViewPaper 메소드 & BookListViewElectronic 메소드

종이책이나 전자책의 목록을 책의 분류에 따라서 보여주는 메소드이다. 이 메소드에서 생성된 각 객체의 가장 최상단의 부모 클래스인 `Book` 클래스의 `BookList()`를 호출했는데, 생성된 객체별로 전부 선언을 하다보니 중복 코드가 많이 생기는 것 같고, 코드가 길어지는 느낌을 받아 상속을 이렇게 쓰는것이 맞는지 의문이 들었다.

#### UserSelectWork() 메소드

도서관 시스템 중 원하는 업무를 선택하는 메소드이다. 

---

### BookLoan

- `ReturnPeriod` 변수는 변하지 않는 값이기 때문에, `static`으로 선언

#### BookLoan 메소드

대여할 책의 권수를 입력하고,  대여할 책의 이름을 입력하여 책을 대출하는 메소드이다. 책의 반납일자는 지정된 날 이내라는 문구를 출력한다. 

---

### 비고

메소드를 구현하면서 `switch` 문을 많이 사용했는데, `default` 문에서 예외 처리를 위해 함수를 다시 한번 호출하는 방식으로 구현했는데, 다음에는 자바에 있는 `try-catch`문을 사용해서 예외 처리를 할 수 있지 않을까 생각이 들었다.
